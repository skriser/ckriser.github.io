requests应该是最pythonic的代表之一。这篇是v0.3.0版本阅读记录

0x01

学习前人阅读方式，先从 HISTORY.rst 开始咯。

在v0.3.0中新增功能：

  1. 自动认证的API做了调整；
  2. 更智能的查询参数设置；
  3. 允许post数据的同时进行文件上传；
  4. 新的认证管理系统

0X02

先来看下自动认证：

原来的auth只有一个简单的类，和几个对AuthObject类操作的函数

小知识:对验证添加过的后期可以用，可以使用全局变量进行缓存

改进AuthManager对autoauth进行专门管理，authManger使用单例，AuthObject也添加了不同的authHandler,取消了全局变量AUTOAUTHS的使用，改用auth_manager = AuthManager()这样在加载这个文件的时候也会使得auth_manager对象常驻内存。因为它是core.py的引用

首先对单例进行分解学习，对AuthManager的创建过程进行debug：

可以看到，三个对象地址都是一致的。应该是cls地址不变，cls.__singleton__保留了对象的引用。这样每次都访问的是相同的对象。

0X03

authManger引入后对_get_opener对应修改

why use **kwargs.pop()

看到作者另一个比较有意思的代码，作者对get,post等操作抽象出来一个request函数，这个函数只有前两个参数是明确的，后面使用了关键字参数。但是，获取数据为甚要用pop而不直接用get呢。

在网上搜了一些，另一个github找到了这样写的想法：why use **kwargs.pop()

With kwargs.pop(), I can verify that the caller isn’t passing in invalid params, e.g., # 有了这个，我可以验证调用者没有传入无效的参数

不过这样写的时候需要通过docstring记录可接受的params，否则使用者会不知道要传哪些
